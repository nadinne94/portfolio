Projeto principal de portfólio

Pipeline Lakehouse de Dados Abertos da Câmara dos Deputados

Plataforma end-to-end de Engenharia de Dados para transformar dados públicos legislativos em produto analítico para Power BI, cobrindo ingestão via API, Delta Lake, arquitetura Medalhão, PySpark, NLP/ML, Star Schema, camada de serving e dashboard.

Resumo da documentação

01

Problema de negócio

Os dados legislativos são públicos e ricos, mas estão distribuídos em múltiplos endpoints, com estruturas diferentes e necessidade de tratamento antes da análise.

02

Fonte de dados

Consumo da API pública de Dados Abertos da Câmara, incluindo deputados, partidos, órgãos, proposições, tramitações, votações, votos, eventos e presenças.

03

Ingestão Bronze

Armazena os dados brutos da API com suporte a paginação, ingestão incremental, endpoints dependentes, watermark, metadados técnicos e logs.

04

Camada Silver

Realiza limpeza e padronização: nomes de colunas, tipos de dados, datas, nulos, textos e duplicidades, criando uma base mais confiável.

05

Camada Gold

Adiciona valor analítico com atributos derivados, flags, regras de negócio, classificação de proposições e preparação para modelagem dimensional.

06

NLP e Machine Learning

Classificação temática e jurídica com estratégia híbrida usando regex, dicionários, modelo supervisionado, fallback e rastreamento com MLflow.

07

Modelo dimensional

Criação de Star Schema com dimensões como tempo, deputado, partido, proposição, evento e órgão, além de fatos para proposições, autoria, tramitação, votação, votos, eventos e presença.

08

Serving e Power BI

Publicação das tabelas finais para consultas SQL, exploração analítica e construção do dashboard legislativo no Power BI.

09

Data Quality

Validações declarativas por tabela e camada, incluindo dataset não vazio, colunas obrigatórias, chaves não nulas, unicidade e controle de nulos.

10

Execução

Execução em ordem conceitual: Bronze, Silver, treinamento ML, Gold, Star Schema e Serving, respeitando as dependências entre as camadas.

11

Decisões técnicas

Medallion separa responsabilidades, Delta Lake aumenta confiabilidade, PySpark permite escala, Star Schema facilita BI e regex + ML melhora a classificação textual.

12

Evoluções futuras

Próximas melhorias: CI/CD, aprimoramento da avaliação dos modelos NLP e amadurecimento do pipeline para cenários mais próximos de produção.
