Projeto principal de portfólio

Pipeline Lakehouse de Dados Abertos da Câmara dos Deputados

Plataforma end-to-end de Engenharia de Dados para transformar dados públicos legislativos em produto analítico para Power BI, cobrindo ingestão via API, Delta Lake, arquitetura Medalhão, PySpark, NLP/ML, Star Schema, camada de serving e dashboard.

Resumo do projeto

Visão geral

O projeto foi desenvolvido para simular um fluxo moderno de Engenharia de Dados aplicado ao domínio legislativo brasileiro. Ele transforma dados brutos da API pública da Câmara dos Deputados em uma base analítica organizada, confiável e pronta para exploração em Power BI.

Problema resolvido

Os dados legislativos são públicos e ricos, mas estão distribuídos em múltiplos endpoints, com estruturas diferentes e relações hierárquicas. O pipeline organiza essas informações para responder perguntas sobre proposições, deputados, partidos, tramitações, votações, eventos e presença parlamentar.

Fonte de dados

A fonte principal é a API de Dados Abertos da Câmara dos Deputados. O projeto consome entidades como deputados, partidos, órgãos legislativos, proposições, autores, tramitações, votações, votos, eventos e presença em eventos.

Arquitetura da solução

A solução segue uma arquitetura Lakehouse em camadas, com Bronze para ingestão bruta, Silver para limpeza e padronização, Gold para enriquecimento analítico, Star Schema para modelagem dimensional e Serving para disponibilização das tabelas ao Power BI.

Ingestão e Delta Lake

A camada Bronze lida com ingestão simples, incremental e dependente, incluindo paginação, controle de watermark, metadados técnicos e logs. Os dados são persistidos em Delta Lake, permitindo maior confiabilidade, histórico transacional e compatibilidade com processamento Spark.

Tratamento e enriquecimento

A camada Silver realiza padronização de colunas, tipos, datas, textos, nulos e duplicidades. A camada Gold adiciona valor analítico com atributos derivados, flags, regras de negócio, categorização de votos e preparação para modelagem dimensional.

NLP e Machine Learning

Um dos diferenciais do projeto é a classificação temática e jurídica de proposições. A solução combina regex, dicionários temáticos, modelo supervisionado, fallback para textos ambíguos e MLflow para rastreamento, versionamento e inferência dos modelos.

Modelo dimensional

O projeto cria um Star Schema com dimensões como tempo, deputado, partido, proposição, evento e órgão. Também cria fatos para proposições, autoria, tramitação, votação, votos, eventos e presença, facilitando análises em ferramentas de BI.

Data Quality e observabilidade

O pipeline inclui validações declarativas por tabela e camada, como dataset não vazio, colunas obrigatórias, chaves não nulas, unicidade e limite de nulos. Logs estruturados registram camada, dataset, status, registros processados e timestamp, apoiando rastreabilidade e auditoria.

Resultado final

As tabelas finais são publicadas para consumo analítico em Power BI, permitindo explorar temas legislativos, autores, partidos, votações, tramitações e eventos. O dashboard público demonstra a aplicação prática do pipeline como produto de dados.
